  3. [What happens when we try to store values from a function when it doesn't return any?]

    4. [If a function doesn't return a value, storing the result will assign the value undefined to the variable.]

  4. [How do we separate parameters from each other when creating a function?]

    5. [Parameters are separated by commas in the function definition. ","]

  5. [Explain how parameter names are related during the invocation and creation of a function.]

    6. [During invocation, actual arguments are passed in order to match the formal parameters defined in the function signature. The parameter names act as placeholders for these values within the function body.]

  6. [Explain why a parameter is undefined when a function call is missing a parameter]

    7. [When a function is called without providing a value for a parameter, that parameter will be undefined inside the function.]

  7. [Explain what happens when we use recursive functions.]

    8. [Recursive functions call themselves, allowing for repeated execution of code until a base case is met. This can be useful for solving problems that can be broken down into smaller, similar smaller problems.]
